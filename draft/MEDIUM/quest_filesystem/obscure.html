Très bien. Maintenant, il faut quand même libérer plus d'espace pour copier les mises à jour des plans (un disque supplémentaire est disponible en DLC chez UBeeCloud©). Mais attention, il ne faut pas supprimer trop d’archives ! Vous devez trouver le dossier dont la suppression libèrerait le plus d’espace tout en restant en dessous des 100k bytes supprimés. 


    - nnafgstd/
      - jsdrnd.nan (54037 bytes)
      - hgyqmh.hwu (58622 bytes)
      - fbzha/
        - dlttnv.npm (13643 bytes)
      - gbdjwrx/
        - okbwcr.tdy (38252 bytes)
        - rpsuzz.dbf (39175 bytes)
        - qxqj/
          - jjzq.jzq (8572 bytes)
      - afjefi/
        - gges/
          - bee.bee (80345 bytes)

Dans cet exemple, les dossiers qui occupent moins de 100,000 bytes sont:

- gged => 80345 bytes
- qxqj => 8572 bytes
- gbdjwrx => 38252 + 39175 + 8572 = 85999 bytes
- fbzha => 13643 bytes
- nnafgstd => 54037 + 58622 + 80345 = 196004 bytes => Trop gros !
85999 > 80345 > 13643 > 8572

Le dossier qui libère le plus d’espace est donc gged, avec 80345 bytes tout en respectant la contrainte de 100k bytes.


Entrez la taille du dossier à supprimer.
